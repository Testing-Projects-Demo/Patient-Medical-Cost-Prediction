# 🏥 Patient Medical Cost Prediction

A machine learning web application that predicts patient medical billing costs based on clinical and demographic information. Features a full patient registration system, auto-generated Patient IDs, real clinical medications per diagnosis, and a downloadable A4 hospital invoice.

Built with Python, TensorFlow, Scikit-learn, and Streamlit.

## ⚙️ How to Run

### Step 1 — Clone the repository
```bash
git clone https://github.com/Testing-Projects-Demo/Patient-Medical-Cost-Prediction.git
cd Patient-Medical-Cost-Prediction
```

### Step 2 — Create a virtual environment (recommended)
```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Mac / Linux
source venv/bin/activate
```

### Step 3 — Install dependencies
```bash
pip install -r requirements.txt
```

### Step 4 — Run preprocessing
Creates `processed_data.csv` and `scaler.pkl`
```bash
python data_preprocessing.py
```

### Step 5 — Train the models
Creates `patient_cost_model.keras`, `rf_model.pkl`, `selected_features.pkl`, and all 10 charts in `img/`
```bash
python model_prediction.py
```

### Step 6 — Launch the web app
```bash
streamlit run app.py
```

Then open `http://localhost:8501` in your browser.

## 🧠 What Each File Does

### config.py
Central configuration file for the entire project. All hardcoded values live here — file paths, dataset column values, risk scores, age bins, model hyperparameters, app settings, and the real clinical medication dictionary (`CONDITION_MEDICATIONS`). Change a value here and all scripts update automatically.

### data_preprocessing.py
- Loads and cleans the raw healthcare dataset (55,500 records)
- Removes duplicates, nulls, and negative billing values
- Engineers new features: Length of Stay, Age Group, Risk Score
- Encodes Test Results as numeric (Normal=0, Inconclusive=1, Abnormal=2)
- One-hot encodes categorical columns
- Scales Age, Length of Stay, Risk Score using StandardScaler
- Saves `processed_data.csv` and `scaler.pkl`
- Uses Python `logging` with timestamps instead of print statements
- All logic wrapped in clean functions with docstrings

### model_prediction.py
- Loads processed data and selects top 20 features using SelectKBest
- Trains and compares 5 models:
  - Linear Regression
  - Ridge Regression
  - Random Forest
  - Gradient Boosting
  - Neural Network (TensorFlow/Keras) with Dropout + BatchNormalization
- Uses EarlyStopping and ReduceLROnPlateau for smart training
- Prints MAE, RMSE, R² evaluation metrics on test set
- Saves 10 individual chart PNGs into `img/` folder
- Saves `patient_cost_model.keras`, `rf_model.pkl`, `selected_features.pkl`

### app.py — 4-Page Streamlit App

**Page 1 — Predict Cost**
- Patient registration form with full personal details (name, phone, address)
- Attender / Guardian details (name, relationship, phone, address)
- Auto-generated Patient ID (P001, P002, P003 …) on every submission
- Condition-driven real clinical medications — dropdown updates based on diagnosis
- Drug class badge showing medication category (e.g. Biguanide, ACE inhibitor)
- Predicts estimated treatment cost using trained neural network
- Shows predicted cost with lower and upper confidence range (±MAE)
- Full A4 hospital invoice rendered with all patient and cost details
- **Download / Print PDF button** — saves invoice as PDF via browser print dialog
- Invoice includes: line-item breakdown, totals, signature strip, hospital footer
- Interactive pie chart showing cost breakdown by component
- All patient records saved persistently to `patients.csv`

**Page 2 — Patient Records**
- Search any patient by Patient ID (e.g. P001) — instant lookup
- Shows 3 tabs per patient: Personal Details, Medical Details, Invoice
- Full records table with name and condition filters
- Summary metrics: total patients, average/highest/lowest predicted cost
- Download all records as CSV

**Page 3 — Data Analysis**
- 4 key metrics: total patients, average bill, average stay, age range
- 6 interactive EDA charts: billing distribution, patients by condition, average billing by condition, insurance provider breakdown, age distribution, admission type breakdown
- Raw dataset sample (first 10 rows)

**Page 4 — Model Report**
- All 10 training charts displayed individually from `img/` folder
- Model performance summary table
- Explanation of why R² is near 0 for this dataset

## 🧾 Patient Registration & Invoice System

### Patient ID Generation
Every new patient is automatically assigned a unique ID in the format `P001`, `P002`, `P003` … The ID increments based on the last record in `patients.csv`.

### Persistent Storage
All patient records are saved to `patients.csv` in the project folder. Data persists across all app sessions on the local machine. Fields saved per patient:

| Category | Fields Saved |
|----------|-------------|
| Identity | Patient ID, Registration Date & Time |
| Patient | Name, Age, Gender, Phone, Address |
| Attender | Name, Relationship, Phone, Address |
| Medical | Blood Type, Insurance, Condition, Test Results, Admission Type, Medication, Stay, Room |
| Cost | Predicted Cost, Lower/Upper Estimate, all 5 breakdown components |

### Printable A4 Invoice
The invoice is rendered as a full A4 page (210mm × 297mm) inside an iframe using `st.components.v1.html()` to avoid Streamlit's HTML sanitizer. It includes:
- Hospital header with name, address, GST
- Patient meta strip: name, age, gender, blood type, admission details, insurance, Patient ID
- 5 line-item table: Base Hospitalization, Medical Condition, Length of Stay, Medication, Ancillary Charges
- Totals section: Subtotal, Insurance Adjustment, Total Payable, Estimate Range
- Signature strip: Patient/Attender, Authorised Signatory, Accounts Department
- **Download / Print PDF** — click the button → browser Print dialog → Save as PDF

## 📊 Dataset

- 55,500 patient records
- Features: Age, Gender, Blood Type, Medical Condition, Insurance Provider, Admission Type, Medication, Test Results, Billing Amount
- Medical Conditions: Arthritis, Asthma, Cancer, Diabetes, Hypertension, Obesity
- Insurance Providers: Aetna, Blue Cross, Cigna, Medicare, UnitedHealthcare
- Medications (dataset): Aspirin, Ibuprofen, Lipitor, Paracetamol, Penicillin
- Admission Types: Elective, Emergency, Urgent

> **Note:** The billing amounts in this dataset are synthetically generated with uniform random distribution. As a result, all models produce R² ≈ 0, which is an honest and expected finding — not a code error. This is a known characteristic of synthetic datasets where target values are randomly assigned independently of patient features.

## 📈 Model Results

All 5 models were trained and compared on the test set (8,229 samples). Due to the synthetic nature of the dataset, R² scores are near 0 across all models.

| Model | MAE | RMSE | R² |
|-------|-----|------|----|
| Linear Regression | ₹12,320 | ₹14,230 | -0.0004 |
| Ridge Regression | ₹12,320 | ₹14,230 | -0.0004 |
| **Random Forest** | **₹12,317** | **₹14,229** | **-0.0002** |
| Gradient Boosting | ₹12,317 | ₹14,238 | -0.0015 |
| Neural Network | ₹12,318 | ₹14,240 | -0.0017 |

**Best Model: Random Forest (R² = -0.0002)**
