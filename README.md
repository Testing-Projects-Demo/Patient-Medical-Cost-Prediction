# 🏥 Patient Medical Cost Prediction

A machine learning web application that predicts patient medical billing costs based on clinical and demographic information.

Built with Python, TensorFlow, Scikit-learn, and Streamlit.

## ⚙️ How to Run

### Step 1 — Clone the repository
```bash
git clone https://github.com/Testing-Projects-Demo/Patient-Medical-Cost-Prediction.git
cd Patient-Medical-Cost-Prediction
```

### Step 2 — Create a virtual environment (recommended)
```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Mac / Linux
source venv/bin/activate
```

### Step 3 — Install dependencies
```bash
pip install -r requirements.txt
```

### Step 4 — Run preprocessing
Creates `processed_data.csv` and `scaler.pkl`
```bash
python data_preprocessing.py
```

### Step 5 — Train the models
Creates `patient_cost_model.keras`, `rf_model.pkl`, `selected_features.pkl`, and all 10 charts in `img/`
```bash
python model_prediction.py
```

### Step 6 — Launch the web app
```bash
streamlit run app.py
```

Then open `http://localhost:8501` in your browser.

## 🧠 What Each File Does

### config.py
Central configuration file for the entire project. All hardcoded values live here — file paths, dataset column values, risk scores, age bins, model hyperparameters, and app settings. Change a value here and all 3 scripts update automatically.

### data_preprocessing.py
- Loads and cleans the raw healthcare dataset (55,500 records)
- Removes duplicates, nulls, and negative billing values
- Engineers new features: Length of Stay, Age Group, Risk Score
- Encodes Test Results as numeric (Normal=0, Inconclusive=1, Abnormal=2)
- One-hot encodes categorical columns
- Scales Age, Length of Stay, Risk Score using StandardScaler
- Saves `processed_data.csv` and `scaler.pkl`
- Uses Python `logging` with timestamps instead of print statements
- All logic wrapped in clean functions with docstrings

### model_prediction.py
- Loads processed data and selects top 20 features using SelectKBest
- Trains and compares 5 models:
  - Linear Regression
  - Ridge Regression
  - Random Forest
  - Gradient Boosting
  - Neural Network (TensorFlow/Keras) with Dropout + BatchNormalization
- Uses EarlyStopping and ReduceLROnPlateau for smart training
- Prints MAE, RMSE, R² evaluation metrics on test set
- Saves 10 individual chart PNGs into `img/` folder
- Saves `patient_cost_model.keras`, `rf_model.pkl`, `selected_features.pkl`

### app.py — 3-Page Streamlit App
**Page 1 — Predict Cost**
- Patient input form with all clinical fields
- Predicts estimated treatment cost using trained neural network
- Shows predicted cost with lower and upper confidence range (±MAE)
- Interactive pie chart showing cost breakdown by component
- Prediction history table saved across the session

**Page 2 — Data Analysis**
- 4 key metrics: total patients, average bill, average stay, age range
- 6 interactive EDA charts: billing distribution, patients by condition, average billing by condition, insurance provider breakdown, age distribution, admission type breakdown
- Raw dataset sample (first 10 rows)

**Page 3 — Model Report**
- All 10 training charts displayed individually from `img/` folder
- Model performance summary table
- Explanation of why R² is near 0 for this dataset

## 📊 Dataset

- 55,500 patient records
- Features: Age, Gender, Blood Type, Medical Condition, Insurance Provider, Admission Type, Medication, Test Results, Billing Amount
- Medical Conditions: Arthritis, Asthma, Cancer, Diabetes, Hypertension, Obesity
- Insurance Providers: Aetna, Blue Cross, Cigna, Medicare, UnitedHealthcare
- Medications: Aspirin, Ibuprofen, Lipitor, Paracetamol, Penicillin
- Admission Types: Elective, Emergency, Urgent

> **Note:** The billing amounts in this dataset are synthetically generated with uniform random distribution. As a result, all models produce R² ≈ 0, which is an honest and expected finding — not a code error. This is a known characteristic of synthetic datasets where target values are randomly assigned independently of patient features.

## 📈 Model Results

All 5 models were trained and compared on the test set (8,229 samples). Due to the synthetic nature of the dataset, R² scores are near 0 across all models.

| Model | MAE | RMSE | R² |
|-------|-----|------|----|
| Linear Regression | ₹12,320 | ₹14,230 | -0.0004 |
| Ridge Regression | ₹12,320 | ₹14,230 | -0.0004 |
| **Random Forest** | **₹12,317** | **₹14,229** | **-0.0002** |
| Gradient Boosting | ₹12,317 | ₹14,238 | -0.0015 |
| Neural Network | ₹12,318 | ₹14,240 | -0.0017 |

**Best Model: Random Forest (R² = -0.0002)**
